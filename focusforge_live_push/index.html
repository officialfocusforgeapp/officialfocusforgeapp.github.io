Accessibility

Accessibility is used so FocusForge can detect which app is in the foreground and apply the blocking rules the user chose. FocusForge does not use Accessibility to read typed content, messages, or general screen contents.

Usage Access and cloud-backed features

Usage Access is optional and powers app-time insights when enabled. Cloud-backed features may rely on configured providers such as Firebase and Google Play billing. Users can review policies, manage permissions in Android settings, and request deletion of cloud-backed account data.
